CMS or starting from scratch? 07-21-2014, 04:54 PM #1
Hello guys,

Im making a survey for my website. I want to find out if there is still someone who writes websites from scratch or if you prefer CMS. I know that sometimes it depends and if you need to make an online store or if your client needs the website quickly, its better to use CMS. But sometimes its even better to work in a team and write it from scratch.

My friends who are developers prefer CMS in every single case. I dont think that they are real developers because they just customise something what is already done and ready for the customer.. and when I ask them about HTML, they dont know how to answer.. but thats only my opinion and I dont want to insult anyone

So I decided to make the survey about it.

What do you prefer?

I prefer writing from scratch because when there is an error, I can find the mistake faster than in the CMS.

One issue with CMS is that whenever a vulnerability is found, your website is vulnerable too and since there are many script kiddies that use google to find potential targets, your website is likely going to be damaged.

RE: CMS or starting from scratch? 07-21-2014, 07:18 PM #6
When I started programming I only used pure PHP for handling my database. I wrote everything from scratch EVERYTIME. I learned a lot and I understood a lot of things. But then I got a developer position which required me to learn "Drupal".. I kicked and screamed but honestly I'm glad I did it. Using a CMS like Drupal (Joomla and wordpress are pretty close to garbage to me) has made me a quicker and better developer. My code is more industry compliant and adding features is super easy.

If you are building something completely custom maybe see if PHP is worth it. If not use a framework (Another reason to love Drupal is that it basically is a Framework with CMS abilities)..
