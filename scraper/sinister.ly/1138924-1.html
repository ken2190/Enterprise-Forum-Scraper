Basic HTML & CSS Tutorial 06-20-2014, 03:50 PM #1
Hello, [username]

Today I bring you a tutorial/class about HTML & CSS. In this tutorial/class I will cover the basics to creating a simple website.
I will make this as noob friendly as I can, feel free to pm if you need help or just want to talk.

[table]
[row]
[cell]HyperText[/cell]
[cell]Web browsers[/cell]
[cell]Understand HTML[/cell]
[cell]HTML Mark-up[/cell]
[cell]Understand CSS[/cell]
[cell]CSS Mark-up[/cell]
[/row]
[/table]


HyperText

Links that allow a picture or TEXT to link to an external site or content. This can be achived by using the href tag.
Code:
<a href="http://hackcommunty.com">You can put text or an image here.</a>For an image you can use. <img src="img/example.jpg"></img>

Web browsers

Web browsers are what we use to view web pages (duh I knew that). Some examples of web browsers are: Google Chrome, Bring,Yahoo, Opera and Firefox...oh wait ya and Internet explorer the crappiest browser ever made (thanks microsoft).


Understanding HTML

HTML is a tag based web language that allows you to center <center></center>, align <align></align> etc. HTML stands for Hyper Text Mark-up Language, a bit of a mouthful if you ask me. HTML tends to be the first language a web designer learns, its not mandatory to learn first but it is suggested. HTML consists of body part, no it is not a human Mr funny guy. It consist of a <html>, <head>, and <body> tag, they are mandatory for you to create a web page. All the content will go in the <html> tag, if outside the <html> the content may not show. HTML tags tend to be simple, allowing a centered picture, when you get into CSS you can give it exact positioning making CSS the main way of positioning content.


HTML Mark-up

Now I will show you the layout of an HTML document.
Code:
<!DOCTYPE html>
        <head>
            <meta charset="UTF-8">
            <title>My first website</title>
            <meta name="description" content="description of your website">
            <meta name="viewport" content="width=device-width">
            <link rel="stylesheet" href="css/style.css">
        </head>
        <body>
            <div id="container">
              <nav id="topnav">
                <a href="index.html">HOME</a>
                <a href="about.html">ABOUT</a>
                <a href="menu.html">MENU</a>
                <a href="about.html">ABOUT</a>
              </nav>
            <div id="content">
              <h1>Welcome to my website</h1>
              <p>This is a short paragraph that can contain any text about your website.</p>
            <figure id="home-image">
              <img src="img/example.jpg" width="500" height="500" alt="Picture with whatever width and height you want">
              <fig caption class="warning">This is a warning.</figcaption>
            </figure>
          </div>
          <footer>
          Copyright yoursite 2014
          </footer>
         </div>
      </body>
    </html>


Understand CSS

CSS stands for Cascading Style Sheet. CSS positions your content in a website but is a whole separate file. To link your CSS you need to use this <link rel="stylesheet" href="css/style.css">. CSS can also control the font size, color and style.


CSS Mark-up
Code:
body {
font-family: 'Lato', sans-serif;
font-size: 18px;
line-height: 24px;
overflow: hidden;
-webkit-text-size-adjust:100%;
}

Thank you for reading my tutorial/class on basic web design.

I'm not really sure how much a new guy would understand/learn from this. There's not really much learning involved with HTML. You can do just fine with a reference sheet.

It's funny how you should the tag's for placing items on the page with HTML (<center> etc.), but you later tell that you use CSS to position the content of the web page?

Also, you're not really telling anyone what version of HTML you're going to teach them. For example, in the beginning of every HTML page, it's recommended to place a DOCTYPE definition, so the browser knows, how to handle the HTML. This is due to the different versions of HTML being used. The declaration you use, are for HTML5, yet you don't tell the user that you're using HTML5. He might look at HTML 4.x and be like; What the hell?

Instead of just telling them about a few tags, and showing them unexplained code. Walk them through creating a website. Show them what happens when you insert a certain tag/CSS code. Explain what the different part does.

By reading this, I wouldn't have any idea of how to create a website. Besides I know that you use HTML, CSS, insert an image, a link and some deprecated ways of aligning/centering content.

Don't take this the wrong way, I'm not trying to be the douche. I just feel like, when you spend so much time on writing an article or a tutorial, it's sad to see it not really teaching the targeted audience anything.
